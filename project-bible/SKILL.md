# Project Bible - 制作圣经生成与加载器

## 项目存储规范

**统一存储目录**：`~/.claude/projects/bibles/`

**文件命名规则**：`{项目名}.bible.md`（项目名使用中文或英文，避免空格，用短横线连接多词）

**目录结构示例**：
```
~/.claude/projects/bibles/
  ├── 天龙传.bible.md         # v3，古风仙侠，进度：第5集
  ├── 星际迷途.bible.md       # v1，科幻，进度：第1集
  └── 暗影刺客-现代篇.bible.md # v2，现代悬疑，进度：第3集
```

---

## 核心规则（必须遵守）

### 0. 模式判断（最优先执行）

| 触发词 | 模式 | 说明 |
|--------|------|------|
| `list`、`列表`、`所有项目`、`我有哪些圣经` | **LIST** | 列出所有项目圣经 |
| `extract`、`提炼`、`生成圣经`、`保存状态`、`会话要结束了`、`上下文快满了` | **EXTRACT** | 从当前会话提炼圣经 |
| `load {名称}`、`加载 {名称}`、`续写 {名称}` | **LOAD** | 按项目名称加载圣经 |
| `load`、`加载`、粘贴圣经内容 | **LOAD** | 直接粘贴内容加载 |
| `update`、`更新圣经`、`合并`、`本集结束`、`增量合并` | **UPDATE** | 增量合并新内容进圣经 |
| `save {名称}` | **SAVE** | 将当前提炼结果保存为指定项目名 |

**模糊时默认规则**：
- 用户粘贴了包含 `# 📖 制作圣经` 的内容 → LOAD 模式
- 用户说"帮我保存/整理"当前内容 → EXTRACT 模式，完成后询问项目名称
- 用户只说"继续" → 询问项目名称，执行 LOAD

## LIST 模式 — 项目圣经管理

### 执行步骤

**Step 1**：读取 `~/.claude/projects/bibles/` 目录下所有 `.bible.md` 文件

```bash
ls -la ~/.claude/projects/bibles/*.bible.md 2>/dev/null || echo "暂无项目"
```

**Step 2**：解析每个文件的元信息（版本/体裁/进度/最后更新），按体裁分组输出：

```
📚 制作圣经项目库｜共 {N} 个项目

🏮 古风/仙侠
  ├── 天龙传          v3 | 第5集/共12集 | 更新于 2025-01-15
  └── 问剑录          v1 | 第1集/共6集  | 更新于 2025-01-10

🚀 科幻/未来
  └── 星际迷途        v2 | 第3集/共8集  | 更新于 2025-01-12

🌆 现代/都市
  └── 暗影刺客-现代篇  v1 | 第2集/共?集  | 更新于 2025-01-08

💡 操作提示：
  - 加载项目：/project-bible load {项目名}
  - 新建项目：/project-bible extract（当前会话内容）
  - 删除项目：删除对应 .bible.md 文件即可
```

**体裁分类标准**：
- 🏮 古风/仙侠/武侠/历史
- 🚀 科幻/未来/赛博/太空
- 🌆 现代/都市/校园/职场
- 🧝 奇幻/魔法/异世界
- 🎭 其他/未分类

## EXTRACT 模式 — 从当前会话提炼圣经

### 执行步骤

**Step 1：询问项目名称**（若用户未指定）：
```
📝 请确认项目名称（用于保存为 {名称}.bible.md）：
  - 如已有项目，输入项目名执行「增量合并」
  - 如是新项目，输入新名称创建新圣经
```

**Step 2：扫描当前会话**，按以下维度逐项提取信息（若某维度无信息则标注「未定义」）：

```
维度1 → 项目基础：项目名称、体裁类型、目标集数/时长
维度2 → 世界观：时代背景、地理空间、核心世界规则、禁忌设定
维度3 → 角色档案：每个角色的外貌Prompt规格（直接可用的描述片段）、性格、与其他角色关系
维度4 → 视觉风格：色调基准、光影偏好、渲染规格（平台/引擎）、参考锚点
维度5 → 运镜规律：惯用景别/运镜组合、禁用运镜、情感弧线类型（爽点/催泪/悬疑等）
维度6 → 音频风格：BGM类型/情绪、已用乐器标签、声效设计
维度7 → 叙事状态：已完成章节摘要（每章≤3句话）、当前情感弧线位置、下一段起点
维度8 → 平台配置：目标平台（即梦/可灵/Runway等）、输出规格
维度9 → 全局负面提示词：所有技能输出都应避免的元素
```

**Step 3：生成圣经文档**，严格按「制作圣经模板」格式输出（见下文）

**Step 4：生成锚定声明**，输出一段 ≤300 字的压缩约束块（供新会话直接粘贴）

**Step 5：提示保存路径**：
```
💾 圣经已生成。保存命令：
   cp 输出内容 → ~/.claude/projects/bibles/{项目名}.bible.md
   
   或直接在 Claude Code 中执行：
   /project-bible save {项目名}
```

## LOAD 模式 — 加载圣经，建立新会话锚定

### 两种加载方式

**方式A：按项目名称加载（推荐）**
```
用户输入：/project-bible load 天龙传
执行：读取 ~/.claude/projects/bibles/天龙传.bible.md
```

**方式B：直接粘贴内容加载**
```
用户粘贴圣经 Markdown 内容
执行：检测 # 📖 制作圣经 标识确认格式，解析内容
```

### 执行步骤

**Step 1：读取圣经**（方式A执行文件读取，方式B解析粘贴内容）

**Step 2：解析关键约束**，提取以下强制约束：
```
① 渲染规格（全局锁定）
② 色调基准 + 全局负面提示词
③ 所有角色外貌规格片段
④ 运镜禁忌列表
⑤ 情感弧线类型 + 当前阶段
⑥ 续写起点
```

**Step 3：输出「会话锚定确认」**：

```
✅ 制作圣经已加载｜{项目名称} v{N}

- 世界观：{概要}
- 视觉规格：{渲染+色调}
- 弧线阶段：{当前位置}
- 角色档案：{N}个角色已加载（{角色名列表}）
- 运镜禁忌：{列表}

⚡ 续写起点：{一句话描述}

🔧 可调用的技能：
```

- 将角色外貌规格自动前置到角色描述
- 将渲染规格和全局负面提示词注入所有 prompt 输出
- 在分镜输出中强制遵守运镜禁忌
- 情感弧线类型保持一致

## UPDATE 模式 — 增量合并圣经

### 核心原则

> **增量合并不是重新生成**：只识别「新增内容」和「明确变更」，不替换已有的稳定内容。

### 执行步骤

**Step 1：确定合并来源**
```
用户输入：/project-bible update {项目名称}
执行：
  - 读取旧版：~/.claude/projects/bibles/{项目名称}.bible.md
  - 分析来源：本次会话中的新生成内容（角色/分镜/场景/音频）
```

**Step 2：按章节执行合并策略**（见下方「分章节合并规则」）

**Step 3：输出「合并报告」**，格式见下文

**Step 4：输出合并后的完整新版圣经**（版本号 +1）

**Step 5：提示保存**：
```
💾 保存新版圣经：~/.claude/projects/bibles/{项目名称}.bible.md（已从 v{N} 升至 v{N+1}）
```

---

### 分章节合并规则（核心）

#### 📋 合并策略总表

| 章节 | 合并策略 | 冲突处理 | 触发条件 |
|------|---------|---------|---------|
| 项目基础（版本/进度） | **替换** | 无冲突 | 每次必更新 |
| 🌍 世界观·时代背景 | **保留旧版** | 新描述与旧描述矛盾时标 ⚠️ | 有新世界设定 |
| 🌍 世界观·核心设定 | **追加新条目** | 重复条目去重 | 有新设定 |
| 🌍 世界规则·✅存在 | **追加（去重）** | — | 有新可用元素 |
| 🌍 世界规则·❌不存在 | **追加（去重）** | — | 有新禁忌元素 |
| 👥 角色·新角色 | **追加完整角色块** | — | 本次出现新角色 |
| 👥 角色·已有角色外貌 | **保留旧版** | 新外貌描述有差异时标 ⚠️ | 角色描述有变化 |

## 会话内约束强化（LOAD后自动启用）

> 以下规则在 LOAD 模式完成后，对本会话内所有技能调用生效：

### 角色一致性铁律
- 每次生成角色相关 prompt 时，**必须**以圣经中该角色的外貌规格片段为基础开头
- **禁止**改变角色的核心视觉标签（发色/瞳色/标志性服装元素）
- 若用户要求「状态变换」（黑化/受伤），在旧规格基础上叠加，不替换

### 视觉风格铁律
- 渲染规格（虚幻5/8K/HDR）为**必填项**，不可省略
- 色调基准和光影描述词作为**必选关键词**出现在所有 scene-prompt / video-prompt 输出中

### 运镜一致性铁律
- 圣经中记录的禁用运镜在本次 storyboard-script 输出中**严格禁用**
- 情感弧线类型**不得与圣经中记录的类型冲突**

### 叙事连贯性铁律
- 续写起点场景作为下一次 storyboard-script / novel-script 的**强制首镜锚点**
- 上一集留下的悬念钩子**必须在本集开头得到呼应**

## 输出质量铁律

| 规则 | 说明 |
|------|------|
| **角色规格必须可直接使用** | 提炼的外貌片段必须能直接粘贴进 character-prompt，不需再描述一遍 |
| **锚定声明必须紧凑** | 「技能锚定声明」块必须 ≤300 字，能在任何会话开头不占太多 token |
| **版本号必须递增** | 每次 extract/update 输出的版本号必须比上次 +1 |
| **叙事摘要必须精简** | 每章摘要 ≤3 句话，聚焦「核心事件+情感节点+悬念状态」 |
| **负面提示词必须中英双语** | 全局负面提示词必须同时提供中文版和英文版 |
| **合并不能丢失旧内容** | UPDATE 模式只追加/替换/标记，不删除任何旧版内容 |
| **冲突必须标记不能静默覆盖** | 任何与旧版矛盾的内容必须用 ⚠️ 标记，等待用户确认 |

### ❌ 去AI感铁律（圣经内容必须通过此自检）

| 禁止 | 错误示例 | 正确方向 |
|------|---------|---------|
| **外貌描述通用化** | "高挑女性，气质出众，五官精致，性格坚强" | 外貌规格必须能区分这个角色和所有其他角色：右眼下一粒小痣，习惯低头时用左手拢发 |
| **叙事摘要平铺** | "第1集：主角出场，发生了一些事，认识了新角色" | 抓核心情感节点：他第一次在山顶看到她，但没有说话——这个细节后来会成为第7集的关键 |
| **色调描述抽象** | "主色调：古典优雅，辅色：神秘深沉" | 必须有颜色：主色调枯荷绿（#4A6741），辅色朱砂红点缀，禁色荧光和纯黑 |
| **世界观空话** | "这是一个充满魔法与危险的奇幻世界" | 具体到规则：魔法需要消耗寿命，每次施法减寿3-7天，这是整个故事的核心矛盾 |
| **弧线位置模糊** | "目前处于发展阶段" | 精确到镜头状态：处于第3集压制段末尾，下集将触发顿悟钩子 |
| **角色关系套话** | "与B角色：对立但相互吸引的关系" | 记录具体事件：她知道他杀了她父亲，但还没有行动——这个"知道"是第2集第4镜 |

> **核心原则**：制作圣经是唯一一份保证整个创作系列不失去灵魂的文档。**每一条记录都必须是只属于这个项目的，不能用在任何其他项目上。**
