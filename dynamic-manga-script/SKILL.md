# 小说改动态漫剧本

将小说文本或小说文档转换为标准动态漫剧本格式，采用交互式工作流逐步引导用户确认集数、场次信息。输出格式参考 `/references/output-example.md` 中的完整剧本示例。

## 动态漫格式标准

动态漫剧本使用五种格式元素：

| 元素 | 格式 | 用途 |
|------|------|------|
| **画面描述** | `△` 开头 | 视觉画面描写，细致可视化 |
| **旁白** | `OV：` 话外音 | 交代整体故事，开场叙述卖点和亮点 |
| **内心独白** | `姓名OS：` | 人物心理活动 |
| **角色对话** | `姓名：` | 人物对白 |
| **场序** | `X-X 天气/场景 地点 主要人物：角色A、角色B` | 场景信息头 |

## 输出格式详解

### 场序行格式

`{集号}-{场号} {天气}/{场景} {地点}  主要人物：{角色1} {角色2}（{关系标注}）`

天气值：`日` / `夜` / `雨夜` / `黄昏` / `晨` 等
场景值：`内`（室内）/ `外`（室外） / `内外`（半室内）
人物间用空格分隔，关系标注如（弟弟）（哥哥）放在人物名后

示例：`1-1 日 内 侯府房间  主要人物：宋妩 沈知非（弟弟）`

### 对话格式

对话中可嵌入动作/情绪/语气描述，放在括号内位于姓名与冒号之间：

`姓名（情绪或动作描述）：对话内容`

示例：`沈知非（俯身贴近，嗓音低沉，唇边笑意邪肆）：夫人，还跑么？`
示例：`宋曦（声音尖锐，带着几分任性的蛮横）：娘亲，我不嫁沈寂安，我要嫁弟弟沈知非！`
示例：`宋母（眉头深锁，指腹轻轻捻着手里的茶盏盖，语气不解）：你不是非要嫁给哥哥么？`

### 内心独白格式

内心独白使用 `OS` 标注，紧接在情绪/动作描述后：`姓名（OS）` 或 `姓名（情绪,OS）`

示例：`宋妩（喘了口粗气，庆幸,OS）：我……我重生了……`
示例：`沈寂安（盯着她看了一瞬，目光沉静，OS）：嗯。宿在书房。`

### 闪回/时间跳转格式

| 标记 | 用法 |
|------|------|
| `【闪回】` | 标记闪回开始 |
| `【闪回结束】` | 闪回结束 |
| `【接1-2】` | 闪回后接回的场次标记 |
| `【梦中闪回】` | 梦境闪回 |
| `【翌日】` | 次日时间跳转 |

### 镜头/画面特殊标记

| 标记 | 用法 | 示例 |
|------|------|------|
| `△【特写】` | 镜头特写 | `△【特写】黄瓜` |
| `△（镜头切至XX）` | 镜头转换 | `△（镜头切至别墅门口）` |

## 小说转剧本的核心规则

### 一、提炼主线，保留精华

- 剧本聚焦围绕主角展开的主线故事
- 去掉或精简支线，同时保持剧情连贯不跳脱
- 保留故事中的爽点、甜点、虐点
- 每次转换只处理1集剧情的文本量
- 爽点/钩子/悬念尽量放在每集末尾

### 二、避免过多文字铺垫

- 不需要过多的情绪、环境描写
- 语言精炼，以台词+画面为主要表现形式
- 背景描写/环境描写/人物心理描写 → 精简或转为台词和画面
- 心理独白 → 外化为可见的细微动作
- 大段环境描写 → 提炼3个视觉层次
- 时间流逝 → 转为视觉符号（蜡烛燃尽/光线变化/翻页/落叶）

### 三、严格忠实于原著（铁律，最高优先级，不可违反）

⚠️ **本规则为最高优先级，优先级高于所有其他规则。违反本规则的输出视为不合格。**

- **对话保真**：保留的台词须和小说原文完全一致，不得改写对白原意、不得添加原文中不存在的话、不得调整对话顺序改变原意
- **情节保真**：只可删减支线，不可自行增加不存在的情节、事件、人物关系、背景设定
- **人物保真**：人物性格、说话风格、行为逻辑不能偏离小说设定（禁止OOC — Out of Character）
- **结局保真**：不得更改任何情节的结局走向和因果关系，不得将原著中未发生的结局提前暗示

**每场输出后必须逐项自检（不合格须修正后重输出）**：
1. 对话是否与原文一致？（逐句对照，不得改写原意）
2. 情节是否有自行添加？（如有，删除并重新输出）
3. 人物是否符合原著性格？（说话方式、决策逻辑、情感反应）
4. 是否有任何原文中没有的剧情元素混入？（如有，立即移除）

**禁止的行为清单**：
- ✖ 自行增加原文没有的对话或内心独白
- ✖ 将原文的叙述性文字改写为角色对话（除非原文明确是对话内容）
- ✖ 让人物说出不符合其性格的话
- ✖ 凭空添加情节转折、冲突、悬念
- ✖ 美化或"优化"原文中的粗糙表达——保留原文的语言风格
- ✖ 将多个人物的台词合并到一个人物身上

### 四、开篇策略

- 用OV在开篇交代整体故事，一开场就将故事的卖点和亮点叙述出来
- 以台词和画面为主，追求爽感，节奏快
- 调整情节的展现顺序，将亮点和核心卖点剧情放在故事开头吸引观众

### 五、合规要求

- 立场不正确、戾气过重的词语要进行规避
- 符合监管法律和小说阅读平台规则

## 交互式工作流

严格按以下6步流程执行，每一步用户确认后才进入下一步：

**第1步**：告知用户"本功能是将小说内容转换为标准动态漫剧本，剧情推进较快，主要保留主线情节，通过台词和画面推进故事"

**第2步**：请用户提供需要改编的小说文本或小说文件路径。提示输入长度尽量控制在1集剧本的体量（参考：小说原文2000-3000字约可输出500-1000字剧本）

**第3步**：请用户确定这段小说改编后是第几集（回复阿拉伯数字）

**第4步**：询问用户是否有场次要求（分为几场，或指定起始场次序号）。若用户无要求，根据剧情内容自行决定场次划分，每集最多5场

**第5步**：严格按上述格式标准将用户输入的小说文本转换为动态漫剧本输出。

**输出前，必须逐场执行以下保真检查（不合格的场次须修正后重输出）**：

1. **对话逐句对照**：将输出中的每一句对话与原文对应位置逐句比对，确认无改写、无添加、无调序
2. **情节完整性**：确认没有自行添加不存在的情节、事件或人物关系
3. **人物OOC检查**：确认每个人物的台词/行为符合原著性格设定
4. **删减标注**：标注所有删减/合并项及其原因（如"精简支线，配角B的对话已压缩"）

输出后询问是否需要调整

**第6步**：告知用户本次转换工作已完成，询问是否要继续下一段转换
- 是 → 跳转第2步
- 否 → 交互结束

## 参考文件

完整的剧本输出示例见 `references/output-example.md`，其中包含3集完整的动态漫剧本，展示了所有格式元素的标准用法。

仅在以下情况读取 `references/output-example.md`：
1. 首次使用本技能且不熟悉动态漫剧本格式
2. 遇到模糊的格式边界（如闪回嵌套、多人对话分层、特殊镜头标记）需要参照示例
3. 用户明确要求对照示例格式

一般情况下，以下内联的"格式速查卡"已足够覆盖常规格式需求，无需每次加载完整3集示例文件（约180行）。

### 格式速查卡（内联，无需外读）

**场序行**：`{集号}-{场号} {天气} {场景} {地点}  主要人物：{角色A} {角色B}（{关系}）`
- 天气：日/夜/雨夜/黄昏/晨 | 场景：内/外/内外

**画面描述**：`△` 开头，每行一个画面单元
**旁白**：`OV：` 话外音
**内心独白**：`姓名OS：` 或 `姓名（情绪,OS）：`
**角色对话**：`姓名（情绪或动作描述）：对话内容`

**闪回标记**：`【闪回】` 开始 → `【闪回结束】` → `【接X-X】` 接回
**梦境闪回**：`【梦中闪回】` | **次日跳转**：`【翌日】`

**特殊镜头**：`△【特写】` 物体 | `△（镜头切至XX）` 转场

## 可调用命令

当用户以自然语言提出小说转剧本请求时，直接执行上述工作流。
用户也可使用 `/dynamic-manga-script` 触发本技能。

## 现有技能关系

- `novel-script` 技能提供更全面的商用级小说转剧本流程（含xlsx导出、角色造型卡、向量知识库等扩展功能）
- 本 `dynamic-manga-script` 技能专注于严格遵循参考示例的格式输出，适用于需要精确格式控制的场景
- 用户已安装 `novel-script` 时，若本技能更符合需求则优先使用本技能

## 可配合的技能

本技能主要输出剧本正文。用户如需将剧本生成AI图文/视频，可后续配合以下技能：
- `cinematic-character-prompt` - 角色画面提示词
- `nine-panel-video-storyboard` - 九宫格视频分镜
- `scene-prompt` - 场景提示词
- `music-prompt` - 配乐提示词

完成剧本转化后建议用户使用 `/project-bible save` 保存当前剧本创作状态，或用 `/project-bible update` 更新项目圣经。
