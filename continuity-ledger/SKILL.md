# Continuity Ledger — 跨镜连续性状态管理器

## Core Use

维护场景级连续性账本——追踪角色位置/身体状态/道具/视线/情绪残留/屏幕方向/摄影机轴线——在每镜设计完成后自动更新，生成承接锚点文本和跨镜一致性告警。

**不处理**: 任何单镜分析——情绪（`$emotion-analysis`）、画面（`$frames-analysis`）、运镜（`$camera-analysis`）、音效（`$audio-design`）、审查（`$content-review`）、提示词汇编（`$ai-prompt-builder`）、多镜工作流编排（`$split-script-to-storyboard`）。

当独立调用时，作为连续性规划工具使用。

## 连续性规则

### 角色管理

- **入场**: 每个入场的角色必须有明确的入场方式——"从画外走进"、"镜头切入"、"原地出现"、"闪回引入"。
- **退场**: 每个退场的角色必须有明确的退场方式——"走向画外"、"被遮挡"、"明确退出空间"、"转为画外音"。禁止静默消失。
- **画外存在**: 在之前镜头中出现且逻辑上仍在空间内的角色，如本镜不可见，必须标注画外位置和遮挡原因。可接受原因：被屏风/门框/柱/其他角色遮挡、站在阴影中、在相邻房间、只闻声不入画。

### 位置追踪

- 每个角色的位置必须相对于场景固定地标描述——不可只用"左边/右边"。
- 位置变化必须在本镜的"核心动作"中有可追溯的移动（走、被拖、被推、站起、坐下）。
- 无原因的位置跳跃 = 连续性错误。

### 道具追踪

- 前镜出现的道具必须在本镜出现，或标注"已放下/已传递/已破坏/被拿走"。
- 道具状态变化：持→放、合→开、完好→破碎、在场→移走。
- 新道具出现：标注"由角色X从画外带入/镜头首次展示"。

### 视线方向

- 对话场景：A看画左 ↔ B看画右（匹配轴线）。
- 单人镜头：视线方向对向画外某人（需标注对谁）。
- 视线变化：本镜中因刺激/震惊/愤怒而改变看向的方向→标注变化触发原因。

### 情绪残留

- 同一场景内，角色情绪不可无原因骤变。
- 情绪变化必须有一镜或多镜的因果推进。
- 情绪跳跃告警：平静→暴怒（中间缺怀疑/确认/震惊等环节）。

### 摄影机轴线

- 同场景对话镜头摄影机统一放置在轴线同侧。
- 越轴必须前置插入缓冲镜（全景/空镜/角色移动）。
- 轴线侧标注：{角色A在画左/角色B在画右，轴线方向从左至右}。

## 四镜窗口审计规则

每 4 个镜头执行一次全窗口审计：

```
检查项:
1. 窗口内角色列表 = 所有在窗口内至少出场一次的角色
2. 对每个角色：
   - 如果镜{N-3}在场且镜{N}在场，中间是否连续 → 检查{N-2}和{N-1}状态
   - 如果镜{N-3}在场但镜{N}不在场 → 是否在窗口内某个镜标注了退场
   - 如果镜{N-2}退场但镜{N-1}又出现 → 是否标注了重新入场
3. 画外角色的位置描述是否跨镜自洽
4. 窗口内是否有三个连续镜头使用相同景别
```

告警格式:
```
⚠️ 四镜审计 - 镜{N-3}-{N}:
  - {角色ID}从镜{N-2}退场后，在镜{N}再次出现，但未标注重新入场方式
  - {角色ID}在镜{N-3}至{N}窗口内从未可见，但末镜仍列为"在场"
```

## 独立使用（非分镜上下文）

当作为独立技能调用时（用户直接 `/continuity-ledger`）:

1. 请用户提供：场景固定地标、初始角色状态、每镜的角色/道具/摄影机变化描述。
2. 用户可以传入已有的分镜序列，本技能自动解析并生成连续性报告。
3. 如果只传入单镜信息，输出单次账本更新和"交给下镜"锚点（跳过四镜审计）。
4. 用户可只要求"四镜审计"（传入4个镜头的数据，输出审计结果）。
