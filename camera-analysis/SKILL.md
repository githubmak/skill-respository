## PIPELINE KERNEL: 运镜景别核心规则

### 景别层级（8级标准）
大远景(人物≤10%)→远景→全景→中景→中近景→近景(胸线上40-60%)→特写→大特写(面部70-90%单镜≤4s)
- 相邻镜景别禁止差一级。允许同景别延续或跨≥2级切换
- 跳级切换允许但必须标注动机（全景→特写=震惊）

### 角度
平视/俯拍(15°/30°/45°/60°)/仰拍(15°/30°/45°/60°)/倾斜(5-15°Dutch angle)/鸟瞰/低角度仰视

**机位角度多样性硬约束（AI视频模型需要视觉变化）：**
- 连续3个以上子镜头不得使用完全相同的机位方向。
- 同一场景内，机位必须在以下选项中轮换：正前方、左前方45°、右前方45°、侧面、过肩、俯角微侧。

- 全景/建立镜头可使用正前方，但后续子镜头必须切换角度。### 焦段
广角14-35mm(环境/多人) | 标准35-70mm(对话/自然透视) | 中长焦70-135mm(特写/压缩空间) | 长焦135-200mm+(远景/偷窥感)

# Camera Analysis — 单镜运镜景别分析器

## AI Video Agent Mode 输出覆盖规则

当由 `ai-video-agent-mode` 作为 Phase 2c 子Agent调用，并收到 dispatch packet 路径时，本技能必须服从调用方 packet，不使用下方独立 Markdown Output Contract。

- 读取 packet JSON。
- `packet.contract_version` 必须为 `modec-v4`；缺失或版本不符时停止并要求主 Agent 重新派发。
- 只处理 `packet.items` 列出的子镜头。
- 输出写入 `packet._batch_output_path`；禁止写公共 `packet.output_path`。
- 根对象必须为 `{"items": [...]}`。
- 每个输入 `subshot_id` 必须且只能对应一个输出 item。
- 每个 item 必须包含 `shot_id` 和 `subshot_id`。
- 不要在聊天回复中粘贴完整 JSON，只报告写入完成和阻塞问题。

本覆盖规则优先于上方 Pipeline Kernel 和下方全部独立调用规则。Mode C v4 中：

- 每子镜只选一种主要运镜；同方向的轻微收束只算同一运镜的落幅。连续互动的一次因果注意力交接按本节后方例外处理，不算第二种运镜。
- 相邻镜头允许保持同景别、同机位或固定镜头；只有信息层级、戏剧重音或空间关系变化才调整，不执行景别/角度多样性配额。
- 焦距、速度、角度和距离只保留 1–2 个决定性锚点；不为凑“量化”堆参数，不使用 XYZ 虚拟坐标。
- 推镜通常不超过 0.3m/s、拉远通常不超过 0.2m/s；超速必须有剧情动机并拆解可执行落幅。
- 机位与人物朝向分离；原文未授权时角色不得直视镜头。
- 打斗/多人镜只围绕一个连续主编舞链或整体戏剧目标设计镜头；同一轨迹内可包含 1–3 个因果相接的接触节拍。连续互动允许一次由台词/动作触发的注意力交接，三选一使用“固定+一次拉焦”“一次单向摇/移重构图”或“演员走位+固定机位”。第二个摄影机方向、第二个独立戏剧焦点、反复抢焦或第二条无关动作链才拆镜。
- 光位仅继承 Scene Agent 已确认的主光关系，不强制侧后逆光、轮廓光或“万能光位”。

## 画幅规则

### 9:16 Vertical

| Dimension | Rule |
|---|---|
| **Shot-size priority** | 大特写 > 特写 > 近景 > 中近景 > 胸像 |
| **Camera movement** | Vertical push/pull, tilt up/down, foot-to-face reveal, small central-axis half-orbit. Y-axis dominant, small amplitude |
| **Forbidden** | Large left-right pans, long horizontal tracking, wide horizontal slides, broad circular moves |
| **Medium shots** | One person or tight two-person tension only. Multi-person → depth stacking |

### 16:9 Horizontal

| Dimension | Rule |
|---|---|
| **Shot-size priority** | 中景 > 中全景 > 全景 > 远景 > 大远景. CU for emotional punctuation only |
| **Camera movement** | Left-right pan, horizontal tracking, dolly, stable follow, horizontal push/pull, 180/360 orbit, compound moves |
| **Medium shots** | 1-4 people, dialogue, confrontation, blocking change |

## 景别层级标准

| 景别 | 英文 | 画面占比 | 适用场景 | 漫剧优先级 |
|---|---|---|---|---|
| 面部特写 | ECU | 面部占70%-90% | 极致情绪、关键台词、微表情 | **仅情绪爆发节点使用** |
| 近景 | CU | 胸线以上，面部40%-60% | 对话博弈、情感表达 | 常规对话镜头 |
| **中近景/中景** | **MCU/MS** | 膝或腰以上 | 社交互动、多人关系 | **漫剧主力景别（叙事最强）** |
| 全景 | FS/LS | 全身可见 | 空间展示、氛围渲染 | 环境建立/过渡 |
| 大远景 | ELS | 人物≤10% | 宏大场景、压抑/孤独叙事 | 少用，仅开场/收尾 |

### 漫剧景别策略
1. 主力景别：中近景（MCU）和中景（MS）占全片≥50%
2. 少用极端角度——除非情绪夸张节点
3. 大特写克制：ECU仅用于核心情绪爆发，全片≤10%
4. 相邻两镜不可同景别（≥1级差异），但光影/色调全程统一

## 焦段功能

- **18-28mm广角**：拉开空间，增强现场感；注意防止人物夸张变形

## 180度轴线规则

### 基本原则
- 同一场景对话分镜摄影机统一放置轴线单侧
- 标注当前轴线侧："A在画左/B在画右，轴线从左至右"

### 越轴许可与过渡方式
越轴可作为创作手段（情感转折、权力反转、空间揭示），但必须通过以下至少一种方式过渡，避免人物位置穿帮或闪现：

| 过渡方式 | 操作 | 适用场景 |
|---------|------|------|
| 缓冲空镜 | 两镜之间插入全景/空镜>=2s，重置空间感知 | 场景切换、情绪大转折 |
| 人物移动越轴 | 角色在镜N末帧转身/绕行至另一侧，镜N+1自然承接新轴线 | 角色主动改变朝向 |
| 机位绕行越轴 | 摄影机从人物后方/头顶绕到轴线另一侧，形成连续跨越 | 对话中的权力反转 |
| 遮挡转场 | 利用遮挡物>=60%画面后切换，实现隐蔽越轴 | 紧张对峙、追逐场景 |

### 越轴时的机位和入画出画调整
当越轴发生时，必须同步调整以下参数避免人物闪现：
- **入画方向**：越轴后角色入画方向须与前镜出画方向承接（如角色从画面右侧出画，则下一镜应从画面左侧入画或已在画面左侧）
- **机位角度**：越轴后机位角度须有明显变化（>=30度差异），避免产生跳跃剪辑感
- **人物位置**：越轴前后角色在画面中的位置（左/右）须保持一致或有可视移动轨迹
- **视线方向**：越轴后角色视线方向须与新的轴线侧一致，不可残留旧轴线视觉惯性

### 越轴机位转换速查
| 越轴前 | 越轴后（推荐） | 过渡方式 |
|-------|-------------|------|
| A画左/B画右 | A画右/B画左 | 插入全景空镜/人物转身 |
| 正前方机位 | 正后方机位 | 机位绕行180度或插入空镜 |
| 45度左前 | 45度右前 | 过肩反打过渡或插入中景 |
| 侧面 | 另一侧面 | 人物转身或空镜过渡 |
## 景别梯子门控

Ladder: `大远景 → 远景 → 全景 → 中景 → 中近景 → 近景 → 特写 → 大特写`
- 相邻景别差一级禁止硬切。允许同景别延续或跨≥2级切换
- 跳级允许（全景→特写=震撼/证据），标注动机
- 同一人物连续3-4镜不能只做景别步进，必须至少有实质故事变化
- 同一角色连续3镜必须包含构图变化或视角切换

## 光位与机位联动

光位方向直接影响人物立体度和机位选择。必须写清精确方向：左前/右前/正侧/侧后/正后方/顶光/底光/45°斜侧——禁止只写"侧面光"。

> 责任划分：光位方向（左前/右前/侧后等）归本技能管——因为光位方向直接约束机位选择。光色/软硬度/强度/色温/专属效果归 `$frames-analysis` 管。

### 光位与机位协同

| 光位 | 适配机位 | 漫剧适用场景 |
|---|---|---|
| **侧后/逆光**（45°侧后方） | 平视/微俯 | **漫剧万能氛围感光**——古风/仙侠/情感慢镜/回忆必用 |
| **45°前侧顺光** | 平视/微俯 | 叙事主镜头、对话特写、表情镜头 |
| **顶光** | 平视/微仰（机位略低于光源） | 吵架、囚禁、悲剧高潮、反派登场 |
| **底光** | 微仰（机位略高于光源） | 惊悚/诡异/恐怖人设/反派特写（少用） |
| **正侧光** | 平视/正对人物 | 对峙、内心挣扎、双面人格 |

### 光位铁律
- 侧后/逆光+人物场景=漫剧万能公式——所有人物镜头优先使用侧后逆光作为轮廓光
- 纯正面平光会扁平→必须45°偏移
- 顶光/底光仅在强戏剧冲突节点使用，全片≤5%镜头
- 光位方向一旦确定，同一场景内不可跳变

## 质量标准（同步自 ai-video-agent-mode）

**景别规范：**
- 仅使用中文景别名称：大特写/特写/中近景/中景/全景/大远景
- 严禁英文缩写：CU/ECU/MS/LS/FS/ELS/WS。check_export.py #12 自动检测，0 容忍

**推镜速度门禁：**
- 推镜 <=0.3m/s，拉远 <=0.2m/s。check_export.py #21 自动检测，超标 >3 次阻断
- 急推/急拉（超速）需要明确的叙事理由并写入 movement_detail

**焦距规则：**
- <=28mm 超广角禁止用于面部特写/大特写。check_export.py #17 自动检测
- 大特写必须使用 >=85mm 焦段

**轴线连续性（check_export.py #18）：**
- 同一场景同组人物相邻镜头保持 180 度轴线。越轴需在 axis_space 中明确过渡理由（角色转身/镜头绕行/空镜转场）
- 特写/大特写轴线标注"不适用（特写镜头，无轴线约束）"
- 逐角色位置解析检测真实翻转，同场景未标注翻转 >5 次阻断

**景别多样：** 同一主镜头内 >=3 个子镜时，景别必须 >=2 种，禁止全相同。
