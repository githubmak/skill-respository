# document-review — 文案审核与校对

## 输入处理

| 输入方式 | 处理 |
|---------|------|
| 直接粘贴文本 | 直接处理 |
| .txt/.md/.docx 文件路径 | 尝试读取；失败提示粘贴 |

## 审核范围

仅检查三类问题：

| 类型 | 检查内容 |
|-----|---------|
| 🐛 剧情bug | 逻辑矛盾、情节不合理、前后不一致、设定冲突 |
| ✏️ 错别字 | 错字、别字、多字、漏字 |
| 📝 语法错误 | 病句、搭配不当、成分残缺、标点误用 |

> **不检查**：写作风格、文采、叙事节奏（除非与设定矛盾）

## 输出格式

### 有问题时

```
🔍 审核结果：发现 N 处问题

🐛 剧情bug

【位置】第X段 / 「...附近文字...」
【原文】<有问题的原文>
【问题】<具体描述>
【修改】<修改后的句子>

✏️ 错别字

【位置】第X段 / 「...附近文字...」
【原文】<有错别字的原文>
【错字】"XXX" → "XXX"
【修改】<修改后的句子>

📝 语法错误

【位置】第X段 / 「...附近文字...」
【原文】<有语法错误的原文>
【问题】<具体错误类型>
【修改】<修改后的句子>

✅ 修正后全文

<应用所有修改后的完整文本>
```

### 无问题时

```
✅ 审核通过，未发现剧情bug、错别字或语法错误。
```

## 知识库：常见错误速查

|---------|---------|------|

| 决对 | 绝对 | 形近混用 |
| 防碍 | 妨碍 | 音近混用 |
| 座落 | 坐落 | 音近混用 |
| 班配 | 般配 | 音近混用 |

| 侯鸟 | 候鸟 | 形近混用 |
| 必竟 | 毕竟 | 音近混用 |
| 按奈 | 按捺 | 音近混用 |

| 辩别 | 辨别 | 形近混用 |

| 痊愈 → 全愈 | 痊愈 | 音近混用 |

### 常见"的/地/得"误用规则

| 规则 | 错误示例 | 正确示例 |
|------|---------|---------|
| 修饰名词用"的" | 他轻轻地手 | 他轻轻的手 |
| 修饰动词用"地" | 她慢慢的走 | 她慢慢地走 |
| 程度补语用"得" | 跑的很快 | 跑得很快 |
| 形容词+动词 | 安静地(坐着)→✓ | — |

### 常见语法病句模式

| 病句类型 | 错误示例 | 修改方式 |
|---------|---------|---------|
| 主谓搭配不当 | "他的心情很孤独" | 改：他感到孤独 |
| 动宾搭配不当 | "完成了目标" | 改：实现了目标 / 完成了任务 |
| 成分残缺 | "通过努力，取得了成功" | 加主语：他通过努力取得了成功 |
| 句式杂糅 | "原因是因为..." | 二选一：原因是.../ 是因为... |
| 否定叠加 | "不能不说没有问题" | 整理逻辑，明确肯/否 |
| 关联词滥用 | "虽然...但是...然而..." | 删除多余关联词 |
| 修饰语歧义 | "我看见他骑着马来了，挥着手" | 谁在挥手？明确主语 |
| 数量词误用 | "一双手套买了三双" | 改：买了三副/三对 |
| 被字句误用 | "这件事被他很快解决了" | 改：这件事很快被他解决了 |
| 时态/时间矛盾 | "昨天他将要出发" | 改：昨天他出发了 |
| 并列不对等 | "他喜欢读书、运动和学习物理" | 并列项应同级：读书/运动/学习 |
| 比较句不完整 | "这篇文章比以前进步了" | 改：比以前的文章进步了 / 比以前进步了 |

### 常见错误标点

| 错误用法 | 正确方式 |
|---------|---------|
| 顿号连接动词："吃、喝、玩" | 动词并列用逗号：吃、喝、玩✓（顿号在动词间也可用） |
| 问句用句号："你去不去。" | 疑问句用问号 |
| 感叹号连用："太好了！！！" | 单用一个感叹号 |
| 省略号3个点："..." | 省略号是6个点："……" |
| 引号内标点外置：「他说"你好"。」 | 引用完整句，句号在引号内：「他说"你好。"」 |
| 书名号括人名：《李白》 | 人名不用书名号，作品才用：《静夜思》 |

### 常见剧情逻辑检查清单

| 检查维度 | 典型问题 |
|---------|---------|
| **时间线** | 年龄/时间前后矛盾、日期跳跃不合理 |
| **空间一致** | 角色从A去B无交代、不可能同时在两处 |
| **人物设定** | 能力/性格前后不一、设定与行为矛盾 |
| **道具追踪** | 道具凭空消失/出现、状态变化无交代 |
| **知识常识** | 违反物理/医学/历史常识 |
| **因果关系** | 行为缺乏动机、结果与原因不匹配 |
| **感官矛盾** | 视觉/听觉描述与场景设定冲突 |
| **天气/环境** | 同场景内天气前后矛盾（晴天→突然下雨无交代）|
| **数量矛盾** | 人数/物品数量前后对不上 |
| **能力边界** | 角色突然拥有之前未提及的技能/知识 |
| **语言设定** | 古代人物用现代词汇/外星人会地球语言但无解释 |

## 标注规则

1. **位置**：精确到段落或用「...附近文字...」定位
2. **原文**：完整引用有问题的原句
3. **修改**：提供可直接替换的正确句子

## 审核质量铁律

| 禁止 | 错误做法 | 正确方向 |
|------|---------|---------|
| **过度解释** | 【问题】这里使用了"的"字，然而根据语法规则修饰动词应该用"地"，所以需要修改 | 【问题】"的/地"误用：修饰动词用"地" |
| **模糊定位** | 【位置】文中某处，大约在中间部分 | 【位置】第3段「慢慢的走」 |
| **凭感觉纠错** | 【问题】这句话读起来有点奇怪，建议重写 | 只标注有明确依据的错误，"奇怪"不是问题 |
| **修改文风** | 作者写"他说道"被改为"他开口说" | 风格偏好不在审核范围内，只改明确错误 |
| **假性完整** | 找不到明显问题时凑出"建议优化""可以考虑" | 直接输出"✅ 审核通过，未发现问题" |
| **漏报比误报危险** | 发现了逻辑矛盾但因为"不确定"就不报 | 逻辑矛盾只要能指出原文具体位置，就应该报 |

> **核心原则**：好的审核让作者立刻知道哪里错了、怎么改，不是让作者读一篇分析报告。**精准 > 完整，确定 > 猜测，简洁 > 详尽。**

## 示例

**输入**：
> 林小明今年十八岁。昨天他刚刚过完十六岁生日。他轻轻的扶她起来。"你没事吧？"他问到。

**输出**：
```
🔍 审核结果：发现 3 处问题

🐛 剧情bug

【位置】第1-2句
【原文】林小明今年十八岁。昨天他刚刚过完十六岁生日。
【问题】年龄矛盾：前文说"十八岁"，后文又说"十六岁生日"
【修改】林小明今年十八岁。昨天他刚刚过完十八岁生日。

✏️ 错别字

【位置】第3句「他问到」
【原文】"你没事吧？"他问到。
【错字】"问到" → "问道"
【修改】"你没事吧？"他问道。

📝 语法错误

【位置】第3句「轻轻的扶」
【原文】他轻轻的扶她起来。
【问题】"的/地"误用：修饰动词应用"地"
【修改】他轻轻地扶她起来。

✅ 修正后全文

林小明今年十八岁。昨天他刚刚过完十八岁生日。他轻轻地扶她起来。"你没事吧？"他问道。
```
