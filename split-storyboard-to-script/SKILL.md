# Split Storyboard to Script

把一行定义为一个**主叙事单元 / 一次 AI 视频生成包**。一行可以包含连续运镜阶段、连续变景别阶段或多个明确剪切的子镜头；不要默认一镜到底，也不要为了“镜头感”机械切景。

## 触发与路由

以下输入直接使用本技能，不再要求用户必须说“转提示词”：

- `镜头拆分`、`拆镜头`、`镜头拆解`、`分镜拆分`
- `把这个大镜头拆成子镜头`、`一行里面设计多个镜头`
- `连续推景怎么拆`、`景别怎么变化`、`镜头1/镜头2/镜头3`
- `把这段镜头整理成Seedance一条多镜头提示词`
- 已提供镜头、场景、分镜表行或视频提示词，要求继续细拆、重组或校验

路由边界：

- 用户已经提供可执行镜头/场景，或目标是AI视频提示词、行内子镜头、时间轴与切换：直接使用本技能
- 用户只提供原始小说/长剧本，要求从零完成完整创意分镜表：先使用 `split-script-to-storyboard`，再回到本技能生成行内子镜头和AI提示词
- 用户说“镜头拆分”但未指定输出形式时，默认输出“主叙事单元 + 时间化子镜头 + 可直接投喂的AI视频提示词”；需要xlsx时再生成五表文件

## 必读引用

执行任务前按需读取：

- 所有任务：完整读取 [timing-validation.md](references/timing-validation.md) 和 [directing-continuity.md](references/directing-continuity.md)
- 目标为 Seedance/即梦或要求单条提示词多镜头：再完整读取 [seedance-prompting.md](references/seedance-prompting.md)
- 生成 xlsx：再完整读取 [output-schema.md](references/output-schema.md)

## 不可违背的优先级

从高到低：

1. 原文剧情、台词、OV、OS与因果完整
2. 人物位置、身体朝向、视线目标、动作方/承受方/阻止方正确
3. 空间轴线、屏幕方向、动作状态、道具和光源连续
4. 台词与子镜头时间窗成立
5. 镜头、景别、运镜、剪辑可执行
6. 光影、材质、情绪和画面美感

镜头设计一旦与人物动作或朝向冲突，必须修改镜头，不得要求人物为了镜头转错方向、换位、跨轴或改变动作目标。

## 工作流

### 0. 强制确认最终导出位置

开始任何读取、分析、镜头拆分、切片或文件生成前，必须先获得用户明确输入的最终导出目录。

- 用户未提供导出位置：只询问“请提供最终文件的导出目录”，暂停后续工作
- 不得自行采用桌面、当前工作区、技能目录、系统临时目录或任何默认路径
- 将用户输入路径解析为绝对路径并回显确认；路径不存在时可在该路径下创建目录
- 用户确认后记录 `export_dir`，本次任务所有写入都必须位于该目录内部

目录约定：

```text
{export_dir}/
├── {项目名}_AI提示词_{画幅}.xlsx
├── prompts/
└── .split-storyboard-cache/
    ├── analysis.json
    ├── slices/
    ├── previews/
    ├── validation/
    └── logs/
```

所有缓存、切片、中间JSON、临时提示词、校验结果和预览必须写入 `{export_dir}/.split-storyboard-cache/`。禁止写入：

- 技能目录及其 `.storyboard-cache`
- `$TEMP`、`%TEMP%`、`/tmp`
- 默认桌面或未获用户授权的工作区目录

如某工具必须接收临时文件路径，也必须在上述缓存目录中创建并传入该路径。任务完成后不擅自删除缓存，除非用户要求清理。

### 1. 导入与原文账本

读取完整脚本/分镜，建立不可遗漏账本：

- 场次、地点、时间与转场
- 出场人物、画外人物、被提及人物
- 每句对白、OV、OS、旁白/字幕及说话人
- 动作触发、反应、结果与核心道具
- 原分镜已有的景别、机位、运镜和声音

导出前逐项回勾。不得因拆镜、合镜或提示词精简删除原文信息；字幕若不要求画面生成，可保留为后期文字轨，但必须在账本中记录。

### 2. 其他 Gates

确认或合理推断：

- 内容风格与渲染风格
- 画幅比例
- 单个主叙事单元最大时长
- 目标平台/模型版本；若用户只说 Seedance，采用通用多镜头结构，不虚构特定版本参数

不要强制写分辨率、帧率、字数上限或负面词分栏，除非用户选择的平台/接口明确支持并需要。

### 3. 全局规范与当前场景前缀

分离两层：

- **全剧规范库**：画风、角色档案、场景档案、色彩策略，只用于管理
- **当前场景执行前缀**：只包含本主叙事单元实际需要的场景、出场人物、服装状态、主光源和氛围

禁止把全剧所有人物、所有场景和所有灯光复制进每条提示词。

每个主叙事单元先列：

```text
【出场人物】
人物A：入画
人物B：画外右侧发声
人物C：仅被提及
```

只列姓名与入画状态，不处理 `Mixed`、主体编号、参考图槽位或资产绑定；用户后续手动绑定资产。

### 4. 主叙事单元划分

优先把同一因果、动作链或情绪链组织在一行，但满足任一条件时拆成新的主叙事单元：

- 场景、时空或主叙事目标改变
- 合并后超过用户时长上限
- 台词无法放进稳定口播阶段
- 主体与动作关系过多，单条提示词无法清晰控制
- 前后状态无法可靠继承

不要按标点机械拆镜；也不要把多个独立动作硬塞进一个长镜头。

### 5. 选择行内组织模式

每一行先选择一种主模式：

#### A. 连续运镜

用于同一机位链、同一动作链、人物位置和朝向持续成立的内容。只安排一个主要摄影机运动；人物复杂动作期间优先简化镜头运动。

#### B. 连续变景别

用于人物停稳后，从环境/身体/手部沿明确视觉路径推进到面部或关键物件。允许多个时间阶段，但仍是一条连续镜头。

要求：

- 起始景别足以承载完整动作；入场、落座、跪下、递交、阻拦等不得塞进只能看肩颈的特写
- 一次只主要改变景别、焦点或观察高度之一
- 不同时叠加环绕、急摇、升降、大幅转身和长对白
- 台词尽量在最终稳定的面部近景/特写阶段开始

#### C. 明确剪切

用于主焦、视角、信息功能、时空或反应主体改变。写清硬切、匹配剪切、视线剪切、动作剪切或叠化，并写出切前结束状态与切后继承状态。

切换必须提供新信息、强化情绪、展示反应、突出道具或解决景别冲突。没有功能的重复景别不得切。

#### 禁止伪剪切

若标题写“镜头1/镜头2”，正文却只是“镜头继续后退/继续推进”，且没有新机位、新焦点或新信息，应改为同一镜头的阶段1/阶段2。禁止用高速变焦、人物融化或场景重构代替剪切。

### 6. 时间轴

一行含多个子镜头/阶段时必须写：

```text
【总时长】10.0秒
0.0-3.5秒｜子镜头1｜近景｜稳定后拉
3.5秒｜匹配剪切；继承上一镜停步状态
3.5-5.8秒｜子镜头2阶段A｜身体细节特写
5.8-7.0秒｜连续上移并轻推
7.0-10.0秒｜面部特写稳定后说台词
```

强制满足：

- 子镜头/画面阶段时间窗连续、无重叠、无空洞；需要多人重叠说话时，在同一画面时间窗内标记重叠对白，不制造重叠镜头时间窗
- 主叙事单元总时长 = 所有子镜头/阶段时长之和
- 每句台词时长 ≤ 它所在的稳定口播时间窗，并留出表演余量
- 环境音、呼吸、脚步、关门、衣料摩擦不按中文口播字数计算

使用 [timing-validation.md](references/timing-validation.md) 的公式，不得只因为口播 ≤ 15秒就标记通过。
组装 JSON 后、生成 xlsx 前运行 `python scripts/validate_timing.py analysis.json`；校验失败时必须修复时间窗，不得绕过。

### 7. 人物、轴线与动作设计

逐子镜头先写人物关系，再设计镜头：

```text
人物位置 → 身体朝向 → 头部朝向 → 视线落点 → 动作方 → 动作路径 → 承受方/目标 → 结果状态
```

要求：

- 画外目标明确写画外左/右/前方；画外人物不得误入画
- 同一连续镜头不允许互斥朝向或双目标动作
- 剪切后逐项复述必要状态，不能只写“保持一致”
- 遵守已建立的180度轴线与屏幕方向；越轴必须通过中性机位、轴上镜头或明确调位建立
- 关键动作的起点、路径、接触/停止位置和结果可见
- 背景虚化人物不得承担精确朝向、攻击、递接或阻拦任务
- 当前说话人的面部和口型必须可见且在焦点内；非说话人不得同步口型
- 景深不能把关键动作目标、接触位置或被注视对象虚化掉
- 镜头路径不得穿过人物、床、墙、桌面等实体，也不得要求无建立镜头的瞬间空间跳跃

发现冲突时依次：简化复合运镜 → 放松景别 → 重排构图 → 明确画外目标 → 改为剪切 → 拆成新的主叙事单元。

### 8. 光影设计

每个当前场景建立可执行的光影连续性：

- 时间与天气
- 主光源位置、方向、软硬、色温倾向
- 辅光/环境反射与人物面部曝光
- 背光、轮廓光、雾、湿地反射等效果的物理来源
- 切镜前后主光方向、阴影方向、肤色和曝光连续

光影必须服务情绪和空间，不得出现无来源的多套主光、日夜冲突、室内外光线混用或为了特写突然换光。特写可提高面部可读性，但不能改变光源逻辑。

### 9. Seedance执行提示词

目标为 Seedance 时，使用自然语言时间轴，让模型明确理解：

1. 当前场景、画风和出场人物
2. 总时长与行内组织模式
3. 跨镜头连续性锁定
4. 按时间顺序排列的子镜头/阶段
5. 转换类型与继承状态
6. 台词、说话人、开始时间和环境音
7. 精简、针对本镜风险的负面约束

Seedance原生具备多镜头叙事能力，但不要把“支持复杂指令”理解成无限复杂。默认每个子镜头只安排一个核心动作和一个主要摄影机运动；一条15秒生成通常控制在2-3个有功能的子镜头，复杂多人动作应更少。详见 [seedance-prompting.md](references/seedance-prompting.md)。

### 10. 自检—修复—提质

逐镜执行，不得只报告问题：

1. **查冲突**：剧情遗漏、时长、位置、朝向、轴线、动作目标、景别承载、运镜、剪切、光影、声音
2. **直接修复**：重写现有字段，直到冲突清零
3. **提质**：再强化构图、材质、微表情、空气感、节奏和关系张力

任何一项未通过，`validation_status = 未通过`，不得进入最终完整提示词表。

## 最终提示词模板

```text
【画幅与风格】{画幅}，{单一主画风与材质}
【当前场景】{空间、时间、陈设、空气与光影}
【出场人物】{姓名：入画/画外发声/仅被提及}
【总时长与组织模式】{X秒；连续运镜/连续变景别/明确剪切/组合}
【连续性锁定】{服装状态、位置、朝向、视线目标、道具、光源、动作起始状态}

{时间窗1}｜{子镜头/阶段}｜{景别、机位、焦点、运镜}
{人物动作、面向、目光、表情、声音}

{转换点}｜{转换类型；切前状态→切后继承状态}

{时间窗2}｜……

【声音】{台词说话人、开始点、口播时长、环境音时间窗、音量层级}
【专项负面】{只针对当前镜头的变脸、动作重置、错朝向、画外人物误入画、剪切被变焦替代等风险}
```

最终提示词必须像可直接执行的视频指令，不得是表格字段的机械拼接。

## xlsx输出

使用 `scripts/generate_xlsx.py` 生成5个工作表。数据结构、字段映射与验证门槛见 [output-schema.md](references/output-schema.md)。

必须显式调用：

```text
python scripts/generate_xlsx.py "{export_dir}/.split-storyboard-cache/analysis.json" --output-dir "{export_dir}"
```

不得省略 `--output-dir`，不得让脚本回退到默认路径。

输出前必须完成：

- 原文账本回勾，台词/OV/OS/旁白无遗漏
- 主镜头、子镜头、动态提示词、完整提示词四层一致
- 时间轴数学检查通过
- 人物位置、朝向、目标、动作路径和轴线检查通过
- 光影与场景连续性检查通过
- 每个非固定镜头有足够起幅/落幅关键画面
- `validation_notes` 明确记录实际核对项和修复，不写空泛“已优化”

只有 `validation_status=通过`、`validation_notes` 非空且 `full_prompt` 为人工整合复检版的镜头，才进入 Sheet 5。
