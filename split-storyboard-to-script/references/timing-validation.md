# 时间计算与校验

## 1. 口播文本提取

只计算实际说出的对白、OV、OS。排除：说话人标签、情绪括号、镜头说明、环境音、动作音和音乐说明。

## 2. 基准语速

- 正常对白/OV：4.5个汉字/秒
- 急促、激动、争吵：5.0个汉字/秒
- 缓慢独白、压抑OS：4.0个汉字/秒

用户提供配音语速或现成音频时，以实际音频时长优先。

## 3. 停顿增量

- `，` +0.10秒
- `。` +0.20秒
- `；` +0.30秒
- `：` +0.20秒
- `？`/`！` +0.20秒
- `、` +0.08秒
- `——` +0.35秒
- `…`/`……` 按一次语义停顿计 +0.25秒，不按字符重复累加

公式：

```text
口播时长 = 实际汉字数 ÷ 对应语速 + 语义停顿总和
```

四舍五入到0.1秒。

## 4. 多人对白

- 顺序说话：各句时长相加，再为每次换人增加0.15-0.35秒反应间隔
- 明确重叠说话：放在同一画面时间窗内，按各声部实际开始/结束时间检查；总体占用取并集，不重复相加，并标记“重叠对白”
- 抢话/打断：前一句只计算实际说出口部分，另加0.10-0.25秒中断反应

## 5. 时间窗门槛

```text
稳定口播阶段时长 >= 口播时长 + 表演余量
```

建议余量：

- 短句：0.3-0.5秒
- 情绪句/说完保留反应：0.5-0.8秒
- 开口前需要抬眼、停顿、吸气：再加相应动作时间

不能只检查口播是否小于主镜头或15秒上限；必须检查它是否小于实际所在子镜头/阶段时间窗。

## 6. 主镜头总时长

- 连续阶段：总时长为各阶段时长之和
- 剪切子镜头：总时长为各子镜头时长之和；剪切点本身不另占时长
- 子镜头/画面阶段时间窗必须首尾相接且不重叠；负时长、空洞均未通过。声音可在同一画面时间窗内重叠
- 环境音、呼吸、脚步、门声等记录声音覆盖时间窗，不按文字字数计算

## 7. 拆分策略

若不合规，按顺序处理：

1. 延长口播所在阶段，但不超过用户上限
2. 压缩不必要的动作或静默
3. 将长台词移到更稳定的面部镜头
4. 按语义与情绪节点拆成新子镜头
5. 仍超限时拆成新的主叙事单元
