# AI Prompt Builder — 单镜AI提示词汇编器


> **Pipeline default: when called by ai-video-agent-mode Phase 6 Composer, MUST use Mode C (director narrative fusion). Mode A/B word limits and ordering rules apply ONLY to standalone use.**

## Core Use

根据上游所有分析器（情绪/画面/运镜/音效/审查）的合并输出，按目标平台的格式规范，汇编四种不同类型的AI生成提示词。

**不处理**: 任何分析——情绪（`$emotion-analysis`）、画面（`$frames-analysis`）、运镜（`$camera-analysis`）、音效（`$audio-design`）、审查（`$content-review`）。本技能仅汇编上游输出为最终提示词文本。也不处理连续性追踪（`$continuity-ledger`）和多镜工作流编排（`$split-script-to-storyboard`）。

当独立调用时，作为提示词格式化工具使用。

---

## 整合流水线总步骤（固定流水线，每镜必走）

```
第1步｜提取分镜基础元数据
  ├── 景别、运镜、时长、画面主体、动作
  ├── 台词/字幕（不混入画面描述，单独输出到字幕框）
  └── 画幅（9:16 / 16:9）

第2步｜拆分画面分层要素
  ├── 人物层、场景层、光影层、画风层、特效层、构图层
  └── 每层独立提取，不跨层混合

第3步｜填充即梦专属参数模块
  ├── 分辨率、帧率、镜头运动、漫剧制式
  └── 负面提示词（独立分栏，不与正向混写）

第4步｜结构化重组语序
  └── 遵循固定顺序：「画风→构图运镜→场景环境→人物主体→动作神态→光影色彩→特效氛围→漫剧专属标签」
  └── 即梦权重逻辑：越靠前的词 AI 识别权重越高

第5步｜匹配即梦生成规则删减/强化
  ├── 剔除 AI 难识别模糊词（好看/唯美/很帅/氛围感强）
  ├── 替换为可视觉化词汇
  ├── 补充漫剧专用关键词
  └── 控制字数区间（静态图≤300字，动态视频≤400字）

第6步｜配套分镜独立约束
  ├── 同系列镜头统一画风、色调、人物设定
  └── 保证漫剧连贯性——只换景别/动作，不改画风/光源/服装

第7步｜区分图文单帧 / 动态漫视频两套模板
  ├── 即梦图片：静态提示词（不写运镜词/帧率/动态参数）
  └── 即梦视频：动态提示词（必须完整写清运镜+速度+视角+时长）
```

## 输入协定

支持两种输入格式。模式 A（全量）为原有格式保留不变；模式 B（精简）为两层分离架构下的高效格式。

### 模式 A：全量输入（原有格式，单镜完整调用）

每个镜号传入所有上游分析器的完整合并输出，按流水线第1-2步提取并分层：

```
镜号: N
画幅: 9:16/16:9
时长: X.Xs

-- 画风层（来自 frames-analysis + 场景级锚定） --
画风类型: {古风/现代二次元/写实漫剧/韩漫/国漫3D渲染/条漫/剧场漫}
线条风格: {干净勾线/厚涂/平涂/水墨笔触/赛璐璐/CG渲染}
质感: {纸张纹理/电影级渲染/8K超清/细腻材质/皮肤质感/布料褶皱}
漫剧标签: {动态漫/有声漫/漫画分镜/剧场版动画/漫画单行本质感}

-- 人物层（来自 emotion-analysis + frames-analysis + 场景级锚定） --
角色锚点: {角色ID+姓名+服装版本}
人物固定模板: {五官/发型/发色/服饰/身高/配饰/肤色/身形}
人物动作: {整体体态+面部神态+手部细微动作，按此顺序}
人物站位: {单人/多人——先主角后配角，标注左右前后}
人物细节: {发丝动态/衣摆飘动/首饰反光/微表情}

-- 场景层（来自 frames-analysis） --
主场景: {环境地点描述}
环境道具: {道具名+位置+状态}
时间天气: {时间+天气+季节}
空间氛围: {室内/室外+环境动态}
场景底色: {连续镜头统一场景参数}

-- 光影层（来自 frames-analysis + emotion-analysis 情绪→光色绑定） --
光源类型: {自然光夕阳/月光/室内烛火/冷色天光/舞台聚光/侧逆光/轮廓光}

软硬度: {软光/硬光/软硬搭配}
光影效果: {柔和漫反射/明暗对比强烈/地面阴影/丁达尔光束/光斑散景/窗棂条状光影/暗角}
色调风格: {莫兰迪冷调/暖橘黄昏/水墨淡青/高饱和国漫/低饱和电影感}

-- 构图层（来自 frames-analysis） --
构图类型: {三分构图/九宫格/对称构图/留白/框架构图/引导线}

-- 特效层（来自 frames-analysis 的专属光影效果 + 动态感加分词） --

特效约束: {缓慢飘落/微弱发光/柔和粒子/动态自然不杂乱}

-- 运镜层（来自 camera-analysis，仅动态视频使用） --
景别: {层级+画面占比}

运镜类型: {固定/推/拉/横移/环绕/跟拍/手持微震/升降}
运镜速度: {缓慢(X.Xm/s)/中速/快速}

-- 连续性层（来自 continuity-ledger） --

输出类型: 静态图片/动态视频

```

### 模式 B：精简输入（两层分离架构，批量高效调用）

当已有全局模板文件时使用。传入字段数量从 ~40 缩减至 ≤15。

```
-- 模板引用 --
全局模板: {文件路径——由 --export-template 生成的模板MD文件}
场景名: {匹配全局模板中的L1场景ID}

-- 镜头标识 --
镜号: S01-XXX
时长: X.Xs

-- L2 镜头级字段 --
景别: {ECU/CU/MCU/MS/FS/LS/ELS}
镜头角度/焦段: {平视/微俯/微仰/俯拍/仰拍}+{mm}
运镜详述: {起始→路径→速度→落定}
构图: {构图描述}
出场角色: {角色ID列表，逗号分隔——须在全局模板角色库中存在}
人物动作与表情: {精简描述——合并时用角色模板+场景光影完整填充}
本镜光影差异: {与场景默认光影的差异——无差异写"无"}
特效: {特效元素——无则写"无"}
台词: {台词+语调标注——无则写"无台词"}
音效: {音效+音乐描述}
连续性: {承接上镜/交给下镜}
违规审查: {✅/⚠️/🚫}

-- 生成参数 --
目标平台: 即梦/可灵/Runway/Pika/Luma/Seedance
输出类型: 静态图片/动态视频
```

**精简输入使用示例**：
```
$ai-prompt-builder --template "全局模板_双胞胎.md" --compact "
场景名: 侯府囚室
镜号: S01-004
时长: 3.5s
景别: CU→CU
镜头角度/焦段: 平视/85mm
运镜详述: 宋妩CU静态1.2s→焦点切沈知非逆光面部CU 1.8s→镜尾静默留白0.5s
构图: 9:16面部特写居中，沈知非半面光半面暗
出场角色: 宋妩,沈知非
人物动作与表情: 宋妩瞳孔收缩针尖+嘴唇发白微颤/沈知非嘴角邪肆笑意不达眼底+眼神沉暗
本镜光影差异: 正面暗部无辅光——极强明暗对比
特效: 无
台词: 沈知非(语气压低→拖长，话中带笑)：夫人，还跑么？【尾音悬停】
音效: 低沉嗓音+心跳渐强→骤停+低频bass沉降+镜尾静默0.5s
连续性: 承接S01-003(铁链绷紧)/交给黑场转场至宋府
违规审查: ✅
"
```

**精简模式的字段覆盖规则**：
- L2 字段若填写，覆盖全局模板中的同名字段
- L2 字段若留空，使用全局模板中匹配的 L1 场景默认值
- `本镜光影差异` 非"无"时，叠加到 L1 场景默认光影参数上
- `出场角色` 必须在全局模板角色库中存在——否则标记 `[模板缺失角色: {ID}]`
- 缺失的 L0 必填字段标记 `[模板缺失: {字段名}]`

## 两层分离架构：全局模板 + 单镜增量

当处理一部剧/一集的大量分镜时，每镜输入包含 ~40 个字段。其中 ~30 个字段在同一部剧内完全不变（画风/角色模板/负面词库）或同一场景内不变（场景描述/光影参数/色彩调性）。为提高效率、降低跨会话恢复成本，本技能支持**两层分离架构**。

### 字段分层

| 层级 | 变更范围 | 内容 | 示例 |
|---|---|---|---|
| **L0 全剧级** | 整部剧不变 | 画幅/渲染引擎/画风前缀/线条风格/质感/漫剧标签/内容风格/全局负面/角色模板(全角色) | `2.5D国漫厚涂渲染` / `【宋妩·v1】墨色长发……` |
| **L1 场景级** | 同场景内不变 | 场景名/主场景描述/环境道具/时间天气/场景光影参数(主光色温+方向+软硬)/色彩调性/构图偏好 | `宋府婚房` / `暖金烛光3000K软光` |
| **L2 镜头级** | 每镜必变 | 镜号/时长/景别/镜头角度+焦段/运镜详述/人物动作+表情/特效/台词/音效/连续性锚点/叙事层级/冲突节点 | `CU面部` / `瞳孔骤缩` / `铁链拖动声` |

### 三种调用模式

#### 模式 A：全量调用（原有模式，保持不变）

传入完整的 40 个字段，直接输出四种提示词。适用于单镜独立调用。

```
$ai-prompt-builder {完整Input Contract字段}
```

#### 模式 B：模板 + 增量调用（新增）

先输出一份全局模板文件，后续逐镜只传 L2 字段 + 模板引用。模板 L0 字段优先填充 → 模板 L1 字段按场景名匹配 → L2 字段覆盖同名字段。

```
# 步骤1：生成全局模板（执行一次）
$ai-prompt-builder --export-template "{剧名}" {完整角色+场景信息}
→ 输出 全局模板.md

# 步骤2：逐镜生成提示词
$ai-prompt-builder --template "{全局模板路径}" --compact "{精简YAML}"
→ 输出该镜四种完整提示词
```

#### 模式 C：分镜表引用调用（新增）

直接读取分镜表 MD 文件中指定镜号的行，自动提取各列数据生成提示词。分镜表可以是 `$split-script-to-storyboard` 的标准输出，也可以是任何包含相同列名的 MD 表格。

```
$ai-prompt-builder --md "{分镜表MD路径}" --shot-id "S01-004"
→ 自动读取表格行，生成四种完整提示词
```

```markdown
光影效果: {丁达尔/暗角/轮廓光等}
色彩调性: {主色调+点缀色}
构图偏好: {9:16垂直居中/三分法等}
曝光值: {EV}
场景特殊规则: {闪回滤镜/特殊色彩偏移等}

### 场景B: {场景名}
...
```

### 精简单镜数据包格式

替代原 ~40 字段的完整 YAML 数据包。精简包只携带 L2 镜头级字段 + 模板引用。`$split-script-to-storyboard` 可在分镜表中用此格式替代原完整数据包。

```yaml
# 精简单镜数据包（≤15个字段）
template: {全局模板文件路径}
scene: {场景名——匹配全局模板中的L1场景}
shot_id: S01-XXX
duration: X.Xs
shot_size: {ECU/CU/MCU/MS/FS/LS/ELS}
lens: {镜头角度/焦段}
movement: {运镜详述——起始→路径→速度→落定}
camera_angle: {平视/微俯/微仰/俯拍/仰拍}
composition: {构图描述}
characters: {本镜出场角色ID列表，逗号分隔——须在全局模板角色模板库中存在}
action_expression: {人物动作+面部表情——精简描述，合并时用角色模板+场景光影完整填充}
lighting_diff: {本镜与场景默认光影的差异——无差异则写"无"}
effects: {特效元素——无则写"无"}
dialogue: {台词+语调标注——无则写"无台词"}
audio: {音效+音乐描述}
continuity: {连续性锚点——承接/交给}
risk: {违规审查状态——✅通过/⚠️需调整/🚫违规}
notes: {执行备注——时长估算等}
```

使用方式：
```
$ai-prompt-builder --template "全局模板_双胞胎.md" --compact "精简YAML"
→ 输出该镜四种完整提示词
```

### 合并规则

当使用 `--template` + `--compact` 模式时：

1. 从全局模板加载 L0 全剧参数和所有 L1 场景模板
2. 按精简包中的 `scene` 字段匹配对应的 L1 场景参数
3. 按 `characters` 列表从全局模板角色库中提取对应角色描述
4. L2 字段覆盖/补充同名的模板字段
5. 缺失的必填字段标记 `[模板缺失: {字段名}]`，不静默补全
6. `lighting_diff` 非"无"时，叠加到场景默认光影参数上（如"正面暗部无辅光"→主光保持但辅光亮度设为0）
7. 按原有模块 1-8 整合流水线，用合并后的完整字段生成四种提示词

### 镜头类型骨架

6 种通用镜头类型骨架，在无完整全局模板时可用。每种骨架预填 L0 提示词前缀 + L1 光影/色彩占位符，单镜调用时只替换 L2 字段：

| 类型 | 骨架名 | 默认画风前缀 | 默认光影 | 适用场景 |
|---|---|---|---|---|
| ① | `对话特写` | 如全局模板 | 软光+正面补光+轮廓光 | 两人对话面部CU/MCU |
| ② | `对峙冲突` | 如全局模板 | 硬光+冷暖对比+暗部压低 | 人物对峙中景/近景 |
| ③ | `OS内心独白` | 如全局模板 | 冷调渐变+暗角+饱和度微降 | 单人面部CU |
| ④ | `环境空镜` | 如全局模板 | 场景主导+大景深+无人物面光 | 全景/远景无人物 |
| ⑤ | `动作打斗` | 如全局模板 | 硬光+动态模糊+速度线 | 大幅肢体动作中景 |
| ⑥ | `闪回记忆` | 如全局模板 | 冷蓝滤镜+饱和度-15%+暗角+微尘 | 任何景别的回忆/过去 |

### 与 `$split-script-to-storyboard` 的接口约定

```
当 split-script-to-storyboard 调用本技能时，采用高效批量模式：

1. 场次开始前 → 生成全局模板（--export-template）
   - 一次性输出 L0+L1，写入文件
2. 每镜 → 使用精简数据包（--template + --compact）
   - 只传 L2 字段，大幅缩减上下文占用
3. 场景切换时 → 新 L1 参数已预置在全局模板中，精简包只需改 scene 字段
4. 关闭会话后恢复 → 加载全局模板文件 + 精简数据包 → 单行命令恢复任意镜号提示词
```

## 模块详细整合规范（逐模块拆解）

### 模块 1：画风整合（最优先写，权重最高，决定整体漫剧质感）

**提取**：从 `frames-analysis` 画风限定词 + 场景描述万能公式的画风维度

**细化补充规则**：

| 画风维度 | 可选词汇 | 选择规则 |
|---|---|---|
| 画风主类 | 古风/现代二次元/写实漫剧/韩漫/国漫3D渲染/条漫/剧场漫 | 整部漫剧全镜头统一，只选1种 |
| 线条 | 干净勾线、厚涂、平涂、水墨笔触、赛璐璐、CG渲染 | 选1种，与主类匹配（古风→厚涂/水墨；现代→平涂/赛璐璐；仙侠→3D渲染） |
| 质感 | 纸张纹理、电影级渲染、8K超清、细腻材质、皮肤质感、布料褶皱 | 最多选2种，8K超清为通用底 |
| 漫剧专属词 | 动态漫、有声漫、漫画分镜、剧场版动画、漫画单行本质感 | 必选1个，区别普通插画 |

**整合规则**：
- 画风放提示词最开头，权重拉高
- 统一整部漫剧所有镜头画风关键词，避免跳画风
- 禁止混合冲突画风：禁止「水墨古风 + 赛璐璐欧美卡通 + 3D写实」混合
- 单镜头只保留 1 种主画风，最多 1 种辅助质感

**整合模板**：
```
{画风主类}，{线条}，{质感}，{漫剧专属词}
例：国漫3D古风渲染，细腻厚涂，8K超清电影级渲染，漫画分镜质感
```

### 模块 2：构图与运镜与景别整合（即梦动态视频核心识别项）

**提取**：从 `camera-analysis` 提取景别/角度/运镜/构图

**标准化改写规则**：

| ❌ 错误写法 | ✅ 完整句式 |
|---|---|
| "近景推" | 中景镜头，缓慢向前推镜，平视视角 |
| "特写环绕" | 面部特写镜头，轻微环绕运镜，平视视角 |
| "全景固定" | 全景镜头，固定机位，微俯视角，三分构图 |

**静态图片 vs 动态视频区分**：

| 项目 | 静态图片 | 动态视频 |
|---|---|---|
| 景别 | ✅ 保留 | ✅ 保留 |
| 构图 | ✅ 保留 | ✅ 保留 |
| 运镜类型 | ❌ 禁止写入（会造成画面模糊扭曲） | ✅ 必须完整标注 |
| 运镜速度 | ❌ 禁止写入 | ✅ 缓慢/中速/快速，精确区分 |
| 视角 | ✅ 保留 | ✅ 保留 |
| 时长 | ❌ 禁止写入 | ✅ 镜头时长5秒/3秒/8秒 |
| 帧率 | ❌ 禁止写入 | ✅ 60帧/30帧 |

**动态视频完整句式**：
```
{景别}，{运镜类型}，{运镜速度}，{视角}视角，{构图类型}，镜头时长{X}秒，流畅动态，无剧烈抖动
例：中景镜头，缓慢向前推镜，平视视角，九宫格构图，镜头时长5秒，流畅动态，无剧烈抖动
```

**避坑**：禁止矛盾镜头描述——不能同时写「固定镜头 + 快速环绕」或「俯拍 + 仰拍」，视角只能保留一种。

### 模块 3：场景环境整合

**提取**：从 `frames-analysis` 的空间分层 + 材质质感词库 + 场景描述万能公式

**整合规则**：
- 分层提取 → 统一重组：`主场景 + 环境道具 + 时间天气 + 空间氛围`
- 原始分镜「古庭院，傍晚，落桃花，石桌灯笼」
- 整合后：「古风庭院，傍晚黄昏，漫天飘落粉色桃花，青石地面，木质石桌，悬挂暖光灯笼，开阔庭院空间」

**环境氛围补充**（从材质质感词库和专属光影效果选取）：
- 雾气、晚风、落叶、星空、室内烛火、闹市人群
- 花瓣缓慢飘落、水面缓缓涟漪、烛火轻轻晃动

**空间约束**：
- 明确区分室内/室外，避免场景冲突
- 连续镜头统一场景底色（同场景同环境参数）
- 漫剧衔接自然——只换景别/动作，不改场景画风

### 模块 4：人物主体整合（防止人物崩坏、变脸）

**提取**：从 `emotion-analysis` 的人设底色 + `frames-analysis` 的人物画面美感描述 + 场景级锚定的角色锚点

**固定人物设定库（整部漫剧通用，每镜必带）**：

```
{角色名}·{人设标签}：{性别}，{发型+发色}，{发饰/发冠}，{服装款式+颜色+材质}，{眉眼特征}，{肤色}，{身形}
```

**示例（男主固定模板）**：
```
少年古装男子，墨色长发束发，白玉发冠，青白色广袖长袍，绸缎反光柔和，眉眼清冷，白皙皮肤，身形挺拔
```

> **铁律**：此模板做成固定前缀，每一段镜头都完整复制，这是避免人物换脸/崩脸的第一关键点。

**动作神态拆分**：
- 语序固定：先整体体态 → 再面部神态 → 最后手部细微动作
- 示例：「身形挺拔而立→眉眼清冷垂眸浅笑→右手轻抬接住飘落桃花」

**多人镜头处理**：
- 先写主角（完整模板），再写配角（简化版）
- 标注人物站位（左/右/前后/远近）
- 禁止写「几个人站一起」——容易人物重叠、肢体穿插畸形

**细节补充**：
- 发丝动态、衣摆飘动、首饰反光、微表情
- 从 `emotion-analysis` 的微表情矩阵提取

**特写镜头**：重点强调面部清晰、五官精致，减少多余背景杂物描述。

**禁忌**：禁止省略关键特征——发色、服饰、配饰不能简写，否则 AI 随机改动人物。

### 模块 5：光影色彩整合（决定漫剧氛围感）

**提取**：从 `frames-analysis` 的精确光色体系 + 光的软硬分级 + 专属光影效果 + `emotion-analysis` 的情绪→光色快速索引

**整合规则**：场景写完紧跟光影，绑定环境：

```
{场景环境}，{光源方向} {光色} {软硬度}为主光源，{辅助光源}，{光影效果}，{色调风格}
```

**示例**：
```
深夜庭院，侧后方冷调冰蓝软月光为主光源，近处灯笼暖橘柔光补面部，柔和轮廓光勾勒人物边缘，地面青石板漫反射细碎冷白光斑，低饱和清冷色调
```

**光源冲突防范**：
- 不能同时写「正午烈日 + 深夜月光」
- 室内烛火场景不要叠加室外强光
- 整部漫剧定主色调，所有镜头沿用
- 前一镜冷色→下一镜高饱和艳色 = 割裂观感，属于不合格

### 模块 6：特效与氛围与动态细节（动态漫加分项）

**提取**：从 `frames-analysis` 的专属光影效果 + 动态感加分词

**标准化整合**：
```
{特效元素}，{速度描述}，{粒子描述}，动态自然不杂乱
```

**示例**：
```
缓慢飘落的桃花花瓣，空气中漂浮细碎月光光斑，柔和粒子特效，动态自然不杂乱
```

**动态视频特效约束**：
- 不要堆砌大量粒子特效——过多光效会导致画面卡顿、人物虚化
- 每镜最多 2 个特效元素，1 个主要 + 1 个次要

### 模块 7：即梦专属参数整合（区分图片/视频）

**（1）即梦静态图片** — 末尾统一加参数：

```
8K，超高清，极致细节，高对比度，精致渲染，漫画分镜，无畸变，五官完整，清晰线条
```

**（2）即梦动态漫视频** — 末尾统一加动态参数：

```
流畅动态，丝滑运镜，60帧，高清动画，人物动作连贯，无闪烁，画面稳定，动态漫标准
```

### 模块 8：负面提示词整合（每镜配套，统一负面库）

**固定通用负面**（所有镜头必带）：

```
畸形五官，扭曲手部，多手指，模糊，低画质，水印，文字，杂乱线条，变脸，人物崩坏，画面撕裂，剧烈抖动，变色
```

**静态图片追加负面**：
```
运动模糊，动态扭曲，帧混合
```

**动态视频追加负面**：
```
画面闪烁，剧烈抖动，人物虚化，帧间跳变，动作卡顿
```

**镜头特殊负面**：
- 室内镜头追加：`无室外强光，无户外天空`
- 特写镜头追加：`无多余杂物遮挡面部，无背景杂乱`
- 多人镜头追加：`无人物重叠，无肢体穿插，无变脸`

### 模块 9：行为负面提示词整合（v11.2约束优先·剧情行为禁令）

> **此模块对应 `$split-script-to-storyboard` v11.2 §0.5 约束系统**。防止 AI 自由发挥导致角色动作崩坏、尊卑错位、目标混淆。

**§9.1 通用行为负面**（所有多人镜必带）：

```
禁止打错目标，禁止情绪错配，禁止角色表情与身份不符，
禁止人物看向错误方向，禁止角色出现在错误空间位置，
禁止角色跨越尊卑边界，禁止景深虚化吞掉人物，
禁止前景遮挡人物面部
```

**§9.2 权谋/宫廷专属行为负面**（古风权谋类必带）：

```
禁止皇后面露暴怒或惊恐，禁止宫女朝皇后/主人抬手，
禁止尊卑角色平起平坐，禁止低位者直视上位者眼睛，
禁止掌掴/击打动作方向指向错误角色，
禁止高位者大幅头部转动或起身前倾
```

**§9.3 行为负面整合规则**：

1. 多人镜（characters≥2）：通用行为负面完整写入 Negative 段
2. 权谋/宫廷类：追加权谋专属行为负面
3. 含力传递动作（掌嘴/扇/打/推/拉/扶/抱等）：追加 `禁止动作错指目标`
4. 极浅景深(F≤2.8)+多人镜：追加 `禁止次要人物面部被景深虚化`
5. 前景含框架元素(烛台/珠帘/纱幔)：追加 `禁止前景遮挡人物面部`

**§9.4 行为负面输出格式**：

```
【行为负面·v11.2约束】
{通用行为负面}
{场景专属行为负面——按 §9.3 规则选取}
```

**负面提示词规则**：
- 独立分栏，不和正向关键词混写——即梦支持正负分离，混写会削弱正向效果
- 禁止输入中文以外杂乱符号、特殊标点、大量换行
- 提示词用逗号分隔，段落简洁

## 整合核心关键点（决定生成效果好坏）

### 1. 人物设定全局统一

整部漫剧建立人物固定描述模板（见 §模块4），所有分镜镜头前置粘贴——这是避免人物换脸/崩脸的第一关键点。

### 2. 语序固定优先级（即梦权重逻辑）

```
画风 > 镜头构图运镜 > 场景 > 人物主体 > 动作神态 > 光影 > 特效 > 参数
```

越靠前的词 AI 识别权重越高。画风、镜头、人物必须放在前半段。所有提示词按此优先级排列，不可调换顺序。

### 3. 文字具象化，拒绝模糊抽象词

| ❌ 禁止 | ✅ 替换为可视觉化词汇 |
|---|---|
| 好看 | 柔和暖光、精致古风服饰 |
| 唯美 | 漫天飞花、柔焦散景、丁达尔光束 |
| 很帅 | 眉眼清冷、身形挺拔、下颌线条分明 |
| 氛围感强 | 低饱和清冷色调、侧逆光轮廓光、朦胧薄雾 |

### 4. 动态镜头区分速度描述

即梦对运镜速度词汇敏感度极高，写错会出现画面抖动/动作过快：

| 速度描述 | 适用运镜 | 画面效果 |
|---|---|---|
| 缓慢（0.2-0.3m/s） | 推镜、拉镜、环绕 | 抒情、情绪积累 |
| 轻微（微震2-5px） | 手持微震 | 紧张、对峙 |
| 中速（0.3-0.6m/s） | 横移、跟拍 | 日常叙事 |
| 快速（0.6-1.0m/s） | 甩镜、快摇 | 动作、转场 |

### 5. 分镜镜头连贯性统一要素

同一场景连续镜头：
- 统一色调、光源、人物服装、环境道具
- 只更换景别、动作
- 不要大幅度改动场景画风

### 6. 长短控制临界点

| 类型 | 字数上限 | 策略 |
|---|---|---|
| 即梦静态图 | ≤300字 | 核心信息前置，次要特效删减压缩 |
| 即梦动态视频 | ≤400字 | 运镜+人物优先，特效最后追加，超长关键词会被截断丢失 |

### 7. 主次分层明确

单镜头有主角、背景、配角时：
- 重点描述主角（占提示词≥50%篇幅）
- 配角简化（姓名+站位+朝向，≤15%篇幅）
- 背景环境简要带过（≤20%篇幅）
- 防止画面元素杂乱无重点

## 整合避坑规则（按类别）

### 画风避坑

- 禁止混合冲突画风：单镜头只保留 1 种主画风，最多 1 种辅助质感
- 漫剧必须加专属标签：「动态漫」「有声漫画」「漫画分镜」——区别普通插画

### 镜头运镜避坑

- 静态图片禁止写入动态运镜词（推镜/环绕/60帧）→ 会造成画面模糊扭曲
- 动态视频必须完整标注运镜 → 缺少运镜 = AI 随机运动
- 视角只能保留一种（禁止「俯拍+仰拍」「固定镜头+快速环绕」）

### 人物整合禁忌

- 禁止省略关键特征：发色/服饰/配饰不能简写
- 多人镜头必须标注站位，禁止「几个人站一起」模糊描述
- 特写镜头减少背景杂物，聚焦面部

### 场景光影防冲突

- 光源时间统一：「正午烈日+深夜月光」= 禁止
- 室内烛火场景禁止叠加室外强光
- 色调跨镜统一：避免前一镜冷色下一镜高饱和艳色

### 即梦平台专属限制

- 禁止输入中文以外杂乱符号、特殊标点、大量换行
- 提示词用逗号分隔，段落简洁，不分段过多
- 台词单独放在字幕框，不混入画面描述——AI 会把文字画进画面
- 负面提示词单独分栏，不与正向关键词混写
- 视频动态漫不堆砌大量粒子特效——过多光效导致画面卡顿、人物虚化

### 信息遗漏补全规则

若分镜缺少光影/画风/细节描述：
- 统一补充整套漫剧通用固定模板
- 不空白留白
- 保证每一条提示词信息完整，无缺失维度

### 漫剧叙事适配

| 镜头类型 | 权重分配 |
|---|---|
| 对话镜头 | 人物面部神态 > 近景特写 > 光影 > 场景（弱化背景） |
| 空镜/环境镜头 | 场景 > 光影 > 远景 > 构图（不写人物） |
| 打斗镜头 | 动态 > 特效 > 广角镜头 > 动作幅度（强化速度感） |

## 平台适配规则

### 即梦静态图片

- 语义理解强，中文直接可用
- 字数上限≤300字——超长截断
- 禁止写入运镜词/帧率/时长 → 会造成画面模糊扭曲
- 负面提示词独立分栏列出
- 逗号分隔，不分段过多
- 台词不混入画面描述

### 即梦动态视频

- 字数上限≤400字——核心信息前置
- 必须完整写清运镜+速度+视角+时长
- 帧率写60帧
- 视频粒子特效≤2个
- 负面提示词追加视频专属项

### 可灵（中文版）

- 更依赖关键词密度
- 重要元素在文首和文末各出现一次
- 画面描述+运镜的语义区分更清晰

### Runway（英文版）

- 使用英文生成，句子简洁
- 负面约束使用 `--no` 前缀
- 运动描述以 `camera` 开头

### Pika / Luma

- Pika：短 prompt 效果更好（<150 words），运动关键词前置
- Luma：长 prompt 效果好，描述越详细越准确

## 冲突解决

当上游分析器之间存在冲突输出时，按以下优先级解决：

1. 技术参数冲突 → `$camera-analysis` 的摄影机决策胜出
2. 画面内容冲突 → `$frames-analysis` 的空间/光影/构图/色彩决策胜出
3. 情绪 vs 画面冲突 → `$emotion-analysis` 的情绪基调胜出，色温值以 `$frames-analysis` 为准
4. 音效与转场影像冲突 → 转场影像优先（影响画面生成）
5. 连续性描述冲突 → `$continuity-ledger` 最新账本为准
6. 即梦平台适配 → 本技能的整合规则胜出（画风语序/字数控制/正负分离/台词不入画面）
7. **模板合并冲突 → L2 镜头字段优先于 L1 场景默认值优先于 L0 全剧参数**——精简单镜数据包的字段显式覆盖全局模板中的同名字段
8. **角色版本冲突 → 若精简包中角色ID在全局模板中存在多个版本，默认取最新版本(vN最大的)；若需指定旧版本，在characters字段写`角色ID·vN`**
9. **v11.2 约束合规冲突 → 若上游分镜输出与约束规则抵触（如F值不统一/坐标缺失/声线错配），标记`[约束冲突: {维度}]`并向上游请求修正——不静默覆盖**

## 与 `$split-script-to-storyboard` 的接口约定

当 `$split-script-to-storyboard` 调用本技能时采用高效批量模式，避免每镜重复传输全局不变字段：

```
1. 分镜设计完成、首镜调用前：
   → $ai-prompt-builder --export-template "{剧名}" {全角色+全场景信息}
   → 一次性生成全局模板文件（含L0全剧参数+L1场景模板+角色模板库）
   → 模板文件保存到与分镜表相同目录

2. 每镜生成提示词时：
   → $ai-prompt-builder --template "{全局模板路径}" --compact "{精简YAML}"
   → 只传 ~15个L2字段，|核心上下文|从 ~2500字/镜降至 ~500字/镜

3. 场景切换时：
   → 精简包scene字段切换为对应的L1场景名
   → L0全剧参数+角色模板库保持不变，无需重新传输

4. 关闭会话后恢复：
   → 加载全局模板文件 + 任意镜号精简数据包
   → $ai-prompt-builder --template "全局模板.md" --compact "{精简YAML}"
   → 一行命令恢复，无需重新读取完整分镜表
```

## 独立使用（非分镜上下文）

当作为独立技能调用时（用户直接 `/ai-prompt-builder`）:

1. 请用户提供：至少画面描述 + 运镜描述 + 目标平台（即梦/可灵/Runway） + 输出类型（静态图片/动态视频）
2. 用户可以只要求某一种提示词类型
3. **单镜快速生成**：用户可先请求生成全局模板（`--export-template`），再逐镜用精简包快速生成
4. **分镜表引用**：用户可提供标准分镜表 MD 文件路径 + 镜号（`--md` + `--shot-id`），直接生成该镜提示词
5. 如果某些字段缺失（如画风/光影/人物模板），按以下默认值补全并标注"[自动补全]"：
   - 画风缺 → 默认为「国漫古风厚涂，8K超清，漫画分镜」
   - 光影缺 → 从时段推断（白天=暖杏柔光，夜晚=冰蓝月光）
   - 人物模板缺 → 警告用户「人物模板缺失可能导致人物不一致，建议提供固定人物设定」
   - 负面提示词 → 始终使用标准负面库
6. 输出语言默认中文，适配即梦格式
7. **当作为 `$split-script-to-storyboard` 的子技能调用时，永远不应用默认值——缺失字段必须标记"[上游缺失]"并向上游请求，不得静默补全。若检测到 v11.2 约束违规（空间锁定缺失/动作目标缺失/声线错配/F值冲突/行为负面缺失），同时标记`[约束冲突: {维度}]`。**
8. **当使用 `--export-template` 生成全局模板时，用户提供的角色信息以完整描述为佳——这是保证跨镜人物一致性的核心**


## 模式 C：导演级叙事融合提示词（短剧全镜完整投喂）

适用场景：Phase 6 Prompt Composer，目标产出为可直接投喂即梦平台的单镜融合提示词。
当由 `ai-video-agent-mode` 调用时，以调用方 `references/format_constraints.md` 为唯一权威；本节只作为单独使用时的简要提醒。

### 核心差异（vs 模式 A/B）

- 字数：遵守 Mode C v4 120-1100 字硬门禁，常规镜头以 220-850 字为软目标。
- 排列：必须显式使用四段 `画面锁定 / 镜头设计 / 表演时间轴 / 光照与声音`，段间一个空行。
- 情绪：先建立 `performance_contract`，再把触发、表情、身体、呼吸/语气、运镜压力和场景压力落实进四段提示词。
- 群像：一个 primary，supporting 只做因果反应，background 作为群体连续层，不逐人分配微反应。
- 连续性：使用 `continuity_contract` 锁定起止、位置、视线、道具、光源和下一镜残留；不新增未确认服装设计。
- 抽卡：使用 `reroll_control` 标记 T2V/I2V/R2V 风险、身份锚点、动作锚点、场景锚点、镜头锚点和参考资产需求。

### 输出结构

每个镜头的 `full_prompt` 必须且只能包含：

1. `画面锁定：` 画幅、风格、人物身份/站位/朝向/服装继承、场景接触与不变项。
2. `镜头设计：` 时长、景别、焦距、机位、轴线、唯一主要运镜和落幅。
3. `表演时间轴：` 2-3 个连续小数秒时间段，写触发、表情控制、肢体动作、台词/OS/OV归属、说话语气、口型边界和终态残留。
4. `光照与声音：` 固定光源关系、环境声和必要声音同步；不重复已出现的原文台词。

### 质量要求

- 表情必须是景别可见的控制，不是名词标签；不强制毫米/角度/次数。
- 不要求群像每人独立微反应；背景群体不得抢主动作。
- 光影只保留能稳定生成的色温、方向和软硬关系，不堆参数。
- 时间分段内表情、动作、语气、镜头落点必须由同一触发因果驱动。
- 可吸收强示例的结构优势：道具动作从指尖、袖口、领口、外套、手机或武器开始，带动肩背/步伐；对白补充句前停顿、音量、语速、气息、咬字或尾音；系统文字只放侧边安全区，不遮脸。示例题材、光线、节奏、场景、服装、角色关系和具体道具事件不得跨项目继承，除非当前剧本或项目配置明确给出。
- 平台适配只吸收可迁移执行原则：正向语序优先“景别/镜头 → 主体站位 → 触发动作 → 表情/肢体递进 → 场景光影 → 必要动态稳定约束”；I2V/R2V 只使用已确认参考资产的真实平台标记锁定身份；复杂动作通过降低速度、幅度、接触节拍和镜头抖动来降抽卡，必要时拆连续片段。

### 输出格式

{"shots": [{"shot_id": "", "subshot_id": "", "duration": 5.0, "full_prompt": "", "negative_prompt": "{{NEGATIVE_PROMPT_AUTO_INJECT}}", "qa_metadata": {"performance_contract": {}, "continuity_contract": {}, "reroll_control": {}}, "generation_control": {}}]}

- C1: 无抽象情绪标签 -> 0
- C2: `performance_contract` / `continuity_contract` / `reroll_control` 齐全 -> 100%
- C3: 合同内容落实到 full_prompt 对应段落 -> 100%
- C4: 表情描述为景别可见控制 -> 0纯名词标签
- C5: 有具体戏剧目标并保存在 QA 元数据 -> 100%
- C6: 有环境音层 -> 100%
- C7: 非说话人口型闭合声明 -> >=90%
- C8: 单镜 120-1100 字，超过必须拆镜 -> 100%
